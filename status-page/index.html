    Status Page Setup¶

    PulseRoute exposes health and metrics endpoints that integrate with external status page providers. This guide covers setting up monitoring for your design partners.

    Health Endpoint¶

    Your PulseRoute API already exposes:

    GET /v1/health

    Response includes: - status: ok, degraded, or unhealthy - engine_running: whether the background evaluation loop is active - tier: current engine tier (1/2/3) - Per-rule health breakdown with processor success rates

    Recommended Setup: UptimeRobot + Statuspage¶

    1. UptimeRobot (free tier)¶

    Create an HTTP monitor pointing at your health endpoint:

    • URL: https://api.pulseroute.dev/v1/health
    • Monitor type: HTTP(s)
    • Interval: 60 seconds
    • Alert contacts: Your team's Slack/email

    Optional: Add a keyword monitor that checks for "status":"ok" in the response body to detect degradation, not just downtime.

    2. Public Status Page¶

    Option A: UptimeRobot Status Page (free)

    UptimeRobot includes a free public status page at stats.uptimerobot.com/your-id. You can CNAME status.pulseroute.dev to it.

    Option B: Atlassian Statuspage (paid, more polished)

    Better for design partners who expect a professional status page. Supports incident management, scheduled maintenance, and component-level status.

    Option C: Instatus or Cachet (self-hosted)

    Lightweight alternatives if you want more control.

    3. Prometheus + Grafana (internal)¶

    PulseRoute exposes Prometheus metrics at /metrics:

    Key metrics to alert on: 

    pulseroute_processor_health           # 1.0=healthy, 0.5=degraded, 0.0=failed
pulseroute_processor_success_rate     # Per-processor success rate
pulseroute_evaluation_errors_total    # Eval loop failures
pulseroute_redis_connected            # Redis connectivity

    The included grafana/ directory has pre-built dashboards. Alert rules to add:

    # Grafana alert: processor degraded for > 5 minutes
- alert: ProcessorDegraded
  expr: pulseroute_processor_health < 1.0
  for: 5m
  labels:
    severity: warning

# Grafana alert: evaluation loop failing
- alert: EvaluationLoopFailing
  expr: rate(pulseroute_evaluation_errors_total[5m]) > 0
  for: 2m
  labels:
    severity: critical

    Webhook Notifications¶

    PulseRoute can notify you when processor status changes:

    # Register a webhook
curl -X POST https://api.pulseroute.dev/v1/webhooks \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-slack-webhook-url",
    "events": ["processor.degraded", "processor.healthy", "failover.triggered", "failover.recovered"]
  }'

    Events are signed with HMAC-SHA256 for verification.

    Recommended Monitoring Stack for Launch¶

    Component Tool Cost
    Uptime monitoring UptimeRobot Free
    Public status page UptimeRobot status page Free
    Internal dashboards Grafana (self-hosted) Free
    Metrics storage Prometheus (self-hosted) Free
    Alerting Grafana alerts + Slack webhook Free

    Total cost: $0. Upgradeable to Datadog/PagerDuty/Statuspage when you have paying customers.
